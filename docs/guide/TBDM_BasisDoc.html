/*! \page TBDM_BASISDoc two-body density matrix on a basis Keyword: TBDM_BASIS \section description Description This object evaluates the one and two-body reduced density matrix on a basis. The basis can be any set of functions that can be expressed in a \ref MO_matrixDoc object. Usually this will be a set of orbitals from a mean-field calculation. The ORBITALS section can often be copied from the .slater file with no changes. The output contains the states section and the density matrices. One thing to keep in mind is that the density matrices are always labeled from 0 to nmo, where nmo is the number of orbitals in the STATES section. The original rating of element i,j is STATES[i],STATES[j], where C/C++ ordering is used. \section example Example 
method { VMC 
  average { 
       TBDM_BASIS
       mode OBDM
       states { 3 4 5 6 } 
       orbitals {
           CUTOFF_MO
           MAGNIFY 1
           NMO 8
           ORBFILE qwalk.orb
           INCLUDE qwalk.basis
           CENTERS { USEATOMS } 
       }
   }
}
include qwalk.sys
trialfunc { include qwalk.slater }
\section options Options \subsection reqopt Required
OptionTypeDescription
ORBITALS/CORBITALS Section Input for a \subpage MO_matrixDoc. When ORBITALS section is used, a real-valued wave function is constructed. When a CORBITALS section is used, the orbitals will be complex-valued.
\subsection optopt Optional While these options are not strictly needed, there are many cases in which the user may want to change them. \subsection expopt Experimental options One shouldn't need to change these options.
OptionTypeDefaultDescription
STATESSection All the orbitals listed in the section ORBITALS A list of the orbitals on which you'd like to evaluate the density matrices.
MODEStringTBDM TBDM: Evaluate the full two-body reduced density matrix
OBDM: Evaluate only the one-body reduced density matrix. This is substantially cheaper
TBDM_DIAGONAL: Evaluate only the diagonal of the two-body reduced density matrix. This is somewhat cheaper and uses les memory than the full TBDM, at the cost of giving less information.
OptionTypeDefaultDescription
NPOINTSInteger 4 Number of auxilliary integration points to use.
NSTEPS_SAMPLEInteger 10 Number of Monte Carlo steps to use to randomize the auxilliary integration points.
*/
